# Вариант 16 — Campus Service (Сервис корпусов)

Сервис управляет справочником корпусов (зданий) учебного заведения: адрес, этажность, контактная информация. **Не хранит** аудитории — они управляются Room Service.

---

## Добавить корпус

Информация, требуемая для создания корпуса:

| Параметр    | Пояснение                  | Обязательность | Тип     | Ограничение                     | Значение по умолчанию |
|-------------|----------------------------|----------------|---------|---------------------------------|-----------------------|
| name        | Название корпуса           | Да             | string  | 1–100 символов, уникально       | —                     |
| address     | Адрес корпуса              | Да             | string  | 5–255 символов                  | —                     |
| city_id     | ID города                  | Да             | integer | существующий id из City         | —                     |
| floors      | Количество этажей          | Да             | integer | ≥ 1                             | —                     |
| description | Описание корпуса           | Нет            | string  | до 500 символов                 | null                  |

**Уникальные комбинации параметров:**
- `name` — название корпуса уникально в пределах всей системы.
- `(address, city_id)` — в одном городе не может быть двух корпусов с одинаковым адресом.

Информация, возвращаемая при успешном создании:

| Параметр    | Тип               |
|-------------|-------------------|
| id          | integer           |
| name        | string            |
| address     | string            |
| city_id     | integer           |
| floors      | integer           |
| description | string или null   |
| is_active   | boolean           |
| created_at  | string (ISO 8601) |
| updated_at  | string (ISO 8601) |

---

## Изменить корпус по ID

Информация, требуемая для изменения (все поля необязательны):

| Параметр    | Пояснение              | Обязательность | Тип     | Ограничение                       |
|-------------|------------------------|----------------|---------|-----------------------------------|
| name        | Новое название         | Нет            | string  | 1–100 символов, уникально         |
| address     | Новый адрес            | Нет            | string  | 5–255 символов                    |
| city_id     | Новый город            | Нет            | integer | существующий id                   |
| floors      | Новое кол-во этажей    | Нет            | integer | ≥ 1                               |
| description | Новое описание         | Нет            | string  | до 500 символов                   |

Информация, возвращаемая при успешном изменении:

| Параметр    | Тип               |
|-------------|-------------------|
| id          | integer           |
| name        | string            |
| address     | string            |
| city_id     | integer           |
| floors      | integer           |
| description | string или null   |
| is_active   | boolean           |
| created_at  | string (ISO 8601) |
| updated_at  | string (ISO 8601) |

---

## Удалить корпус по ID

Вернёт `true`, если корпус был деактивирован (`is_active = false`), иначе `false`. Физически запись из БД не удаляется.

---

| Параметр    | Пояснение              | Тип               |
|-------------|------------------------|-------------------|
| id          | Идентификатор          | integer           |

| address     | Адрес                  | string            |

## Получить список корпусов по заданным параметрам

| Параметр  | Пояснение                    | Тип     |
|-----------|------------------------------|---------|
| city_id   | Фильтр по городу             | integer |

| description | string или null   |

| created_at  | string (ISO 8601) |

## Добавить контакт корпуса

| Параметр        | Пояснение             | Обязательность | Тип     | Ограничение                    |
|-----------------|-----------------------|----------------|---------|--------------------------------|
| campus_id       | ID корпуса            | Да             | integer | существующий id                |
| contact_type_id | ID типа контакта      | Да             | integer | существующий id из ContactType |
| value           | Значение контакта     | Да             | string  | 1–200 символов                 |

**Уникальные комбинации:** `(campus_id, contact_type_id)` — у корпуса может быть только один контакт каждого типа.

Информация, возвращаемая при успешном создании:

| Параметр        | Тип     |
|-----------------|---------|
| id              | integer |
| campus_id       | integer |
| contact_type_id | integer |
| value           | string  |

---

## Удалить контакт корпуса по ID

Вернёт `true`, если запись удалена, иначе `false`.

---

## Получить список контактов корпуса по ID корпуса

| Параметр  | Пояснение  | Тип     |
|-----------|------------|---------|
| campus_id | ID корпуса | integer |

Информация возвращается в виде списка контактов:

| Параметр        | Тип     |
|-----------------|---------|
| id              | integer |
| campus_id       | integer |
| contact_type_id | integer |
| value           | string  |
