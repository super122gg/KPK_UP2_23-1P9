# Вариант №4
# Сервис управления разрешениями (Permission Service)

## Сущность: Permission (Разрешение)

Тонкая настройка прав: кто может редактировать расписание, кто только смотреть, кто может назначать замены.

---

### 1. Информация для создания сущности

`POST /permissions`

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
|----------|-----------|----------------|-----|-------------|-----------------------|
| `name` | Название разрешения | Да | str | длина ≤ 100, уникально | — |
| `description` | Описание разрешения | Нет | str | длина ≤ 255 | `''` |

> **Примечание:** Поле `is_active` при создании автоматически устанавливается в `true`.

**Уникальные комбинации параметров:**
- `name` — должно быть уникальным

### 2. Информация, возвращаемая при успешном создании

| Параметр | Тип |
|----------|-----|
| `id` | int |
| `name` | str |
| `description` | str |
| `is_active` | bool |

---

## Изменить сущность по ID

### 3. Информация для изменения

`PUT /permissions/{id}`

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
|----------|-----------|----------------|-----|-------------|-----------------------|
| `name` | Новое название | Нет | str | длина ≤ 100, уникально | текущее значение |
| `description` | Новое описание | Нет | str | длина ≤ 255 | текущее значение |

> **Примечание:** Поле `is_active` не может быть изменено через данный эндпоинт. Для изменения активности используется soft delete (см. раздел «Удалить сущность по ID»).

### 4. Информация, возвращаемая при успешном изменении

| Параметр | Тип |
|----------|-----|
| `id` | int |
| `name` | str |
| `description` | str |
| `is_active` | bool |

---

## Удалить сущность по ID (soft delete)

`DELETE /permissions/{id}`

> **Важно:** Фактически запись из БД не удаляется, а помечается как неактивная через булевое поле `is_active = false`. При успешном удалении возвращается `True`, иначе `False`.

**Возвращаемое значение:** `True`, если разрешение было закрыто (удалено), иначе `False`

---

## Получить сущность по ID

`GET /permissions/{id}`

### Информация, возвращаемая при успешном поиске

| Параметр | Пояснение | Тип |
|----------|-----------|-----|
| `id` | Уникальный идентификатор разрешения | int |
| `name` | Название разрешения | str |
| `description` | Описание разрешения | str |
| `is_active` | Активность разрешения | bool |

---

## Получить список сущностей по заданным параметрам

`GET /permissions`

### Параметры для получения списка

| Параметр | Пояснение | Тип |
|----------|-----------|-----|
| `name` | Фильтр по названию (частичное совпадение, регистронезависимо) | str |
| `is_active` | Фильтр по активности | bool |
| `limit` | Максимум записей 1000 (по умолчанию 100) | int |
| `offset` | Смещение для пагинации (по умолчанию 0) | int |

### Информация, возвращаемая в виде списка сущностей

| Параметр | Тип |
|----------|-----|
| `id` | int |
| `name` | str |
| `description` | str |
| `is_active` | bool |

## Сущность: RolePermission (Связь роли и разрешения)

Служебная транзитивная таблица для реализации связи **многие ко многим** между ролями (Role Service) и разрешениями (Permission Service).

> **Примечание:** Для сущности RolePermission **не используется soft delete**, так как это служебная таблица. Удаление происходит физически (hard delete).

---

### 1. Информация для создания связи

`POST /role-permissions`

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
|----------|-----------|----------------|-----|-------------|-----------------------|
| `role_id` | ID роли (из Role Service) | Да | int | — | — |
| `permission_id` | ID разрешения (из Permission Service) | Да | int | — | — |

**Уникальные комбинации параметров:**
- `role_id` + `permission_id` — пара должна быть уникальной

### 2. Информация, возвращаемая при успешном создании

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| `id` | int | ID записи связи |
| `role_id` | int | ID роли |
| `permission_id` | int | ID разрешения |

---

## Изменить сущность по ID

`PUT /role-permissions/{id}`

> Для транзитивной таблицы изменение не предусмотрено. Для изменения связи необходимо удалить существующую и создать новую.

### Информация, возвращаемая при попытке изменения

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| `detail` | str | Сообщение об ошибке |

---

## Удалить сущность по ID

`DELETE /role-permissions/{id}`

> **Важно:** Запись физически удаляется из БД (hard delete), так как это служебная транзитивная таблица.

**Возвращаемое значение:** `True`, если связь была удалена, иначе `False`

---

## Получить сущность по ID

`GET /role-permissions/{id}`

### Информация, возвращаемая при успешном поиске

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| `id` | int | ID записи связи |
| `role_id` | int | ID роли |
| `permission_id` | int | ID разрешения |

---

## Получить список сущностей по заданным параметрам

`GET /role-permissions`

### Параметры для получения списка

| Параметр | Пояснение | Тип |
|----------|-----------|-----|
| `role_id` | Фильтр по ID роли | int |
| `permission_id` | Фильтр по ID разрешения | int |
| `limit` | Максимум записей 1000 (по умолчанию 100) | int |
| `offset` | Смещение для пагинации (по умолчанию 0) | int |

### Информация, возвращаемая в виде списка сущностей

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| `id` | int | ID записи связи |
| `role_id` | int | ID роли |
| `permission_id` | int | ID разрешения |

---

## Удалить связь по параметрам (role_id + permission_id)

`DELETE /role-permissions?role_id={role_id}&permission_id={permission_id}`

> **Важно:** Это альтернативный способ удаления связи, не требующий знания `id` записи. Запись физически удаляется из БД (hard delete).

### Параметры для удаления (в Query строке)

| Параметр | Пояснение | Обязательность | Тип |
|----------|-----------|----------------|-----|
| `role_id` | ID роли | Да | int |
| `permission_id` | ID разрешения | Да | int |

**Возвращаемое значение:** `True`, если связь была удалена, иначе `False`

---

## Получить разрешения по ID роли

`GET /role-permissions/by-role/{role_id}`

### Параметры для получения списка

| Параметр | Пояснение | Тип |
|----------|-----------|-----|
| `role_id` | ID роли (передаётся в пути) | int |

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| `id` | int | ID разрешения (из сущности Permission) |
| `name` | str | Название разрешения |
| `description` | str | Описание разрешения |
| `is_active` | bool | Активность разрешения |

## Точки входа REST API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/permissions` | Создать разрешение |
| GET | `/permissions/{id}` | Получить разрешение по ID |
| GET | `/permissions` | Получить список разрешений с фильтрацией |
| PUT | `/permissions/{id}` | Обновить разрешение |
| DELETE | `/permissions/{id}` | Удалить разрешение (soft delete) |
| POST | `/role-permissions` | Создать связь роли и разрешения |
| GET | `/role-permissions/{id}` | Получить связь по ID |
| GET | `/role-permissions` | Получить список связей с фильтрацией |
| PUT | `/role-permissions/{id}` | Изменить связь (не предусмотрено) |
| DELETE | `/role-permissions/{id}` | Удалить связь по ID (hard delete) |
| DELETE | `/role-permissions` | Удалить связь по параметрам (role_id, permission_id) |
| GET | `/role-permissions/by-role/{role_id}` | Получить все разрешения для роли |
